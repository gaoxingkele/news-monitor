## 三、技术实现架构

```
people/data_collection/
├── __init__.py
├── config.py              ← 搜索 API key / 代理 / LLM 配置（复用 .env）
├── models.py              ← SearchTask / SearchResult / ExtractedFact 数据模型
├── search_dispatcher.py   ← 统一搜索调度（Brave+Tavily+Grok+Gemini 异步并发）
├── extractor.py           ← LLM 结构化信息提取（从搜索结果→结构化事实）
├── scrapers/
│   ├── cec_votes.py       ← 中选会选票爬取
│   ├── sunshine_assets.py ← 监察院财产申报爬取
│   ├── sunshine_donation.py ← 监察院政治献金爬取
│   ├── ly_records.py      ← 立法院提案/出席记录爬取
│   └── ccw_rating.py      ← 公督盟评鉴爬取
├── tasks/
│   └── chiang_wanan.py    ← 蒋万安专用采集任务清单（30项）
├── run_collect.py         ← 主入口（按 Tier 分阶段执行）
├── raw/                   ← 原始搜索结果存档
├── extracted/             ← LLM 提取后的结构化 JSON
└── reports/               ← 汇总采集报告
```

### 关键设计决策

1. **搜索调度器**（`search_dispatcher.py`）：
   - 输入：`SearchTask(query_zh, query_en, sources, max_results, time_range)`
   - 对同一任务并发调用 2-3 个搜索引擎，合并去重
   - 输出：`list[SearchResult(title, url, snippet, date, source_engine)]`

2. **LLM 结构化提取**（`extractor.py`）：
   - 输入：`list[SearchResult]` + `extraction_schema`（期望输出的 JSON schema）
   - 用 Grok-fast 或 Qwen 做批量提取（国内源直连，快且便宜）
   - 输出：`list[ExtractedFact(fact, evidence_grade, source_url, date)]`

3. **网络图谱构建**（Tier-2 后处理）：
   - 用 Python `networkx` 从提取的关系数据构建有向图
   - 计算：度中心性、中介中心性、接近中心性、k-core、结构洞
   - 输出：JSON 节点-边格式 + 量化指标表
