# 政治人物系统性分析框架（OSINT）

> **版本**: 2.0
> **适用对象**: 涉及美国—台湾—中国三角关系的政治人物深度画像
> **设计原则**: 事实先行、证据分级、模块化采集、三角联动分析

---

## 第零部分：方法论基础设施

在执行任何模块之前，分析团队须统一以下四项标准。所有模块的输出均须遵循这些标准。

### 0.1 证据分级体系（Evidence Grading Scale）

每条事实性信息须标注以下等级之一：

| 等级 | 定义 | 典型来源示例 |
|------|------|-------------|
| **A — 官方/法定记录** | 由政府机关、法院、立法机关产生的正式文件 | 法院判决书、监察院纠正案文、选举委员会公告、财产申报表、立法院公报、政府公报、公司登记 |
| **B — 权威媒体实名报道** | 由具编辑审核流程的媒体机构产出，含实名记者与可追溯采访对象 | 主流日报/通讯社实名报道、具名专访、公开记者会逐字稿 |
| **C — 可追溯但单一来源** | 可指向具体出处但未经独立交叉验证 | 单一媒体独家报道、政论节目发言、智库报告（非同行评审）、社群媒体官方帐号发文 |
| **D — 弱来源/待核实** | 来源模糊、匿名或无法独立验证 | 匿名爆料、论坛传言、未署名社群截图、二手转述无原始出处 |
| **E — 分析推断** | 非直接事实，由分析者基于已有证据做出的逻辑推断 | 框架内所有"分析题"的结论 |

**使用规则**：
- 每条信息必须同时标注【等级】与【来源链接/引用】与【获取日期】
- A/B 级信息可直接纳入画像结论
- C 级信息须标注"待交叉验证"
- D 级信息仅可作为线索列入附录，不可纳入正文结论
- E 级推断须明确列出所依据的事实编号与推理逻辑

### 0.2 时间分期框架（Periodization Framework）

针对台湾政治人物，建议采用以下标准时间分期，以便不同议题领域的立场变化在同一坐标上对照：

| 代号 | 时期 | 标志事件 |
|------|------|---------|
| P0 | 戒严时期（—1987） | 解严以前的政治社会化阶段 |
| P1 | 民主化初期（1987—1996） | 解严、国会全面改选、总统直选 |
| P2 | 首次政党轮替（1996—2008） | 李登辉后期至陈水扁两任 |
| P3 | 马英九时代（2008—2016） | 二次政党轮替、两岸大交流 |
| P4 | 蔡英文时代（2016—2024） | 含太阳花运动后效应、美中对抗升级 |
| P5 | 赖清德时代（2024— ） | 少数政府、三党不过半 |

**使用规则**：
- 每条立场/表态须标注所属时期代号
- 立场转向分析须至少跨两个时期对照
- 可根据人物生涯增加自定义子时期（如留学期、地方执政期）

### 0.3 题目类型标记（Question Type Tags）

每道题前标注类型，以明确执行方式与产出标准：

| 标记 | 含义 | 执行要求 |
|------|------|---------|
| 📋 **采集** | 事实信息采集 | 须提供可核验来源、证据等级 |
| 🔄 **交叉验证** | 对已采集信息的独立核实 | 须引用不同于首次采集的来源 |
| 🔬 **分析** | 基于事实的推断与评估 | 须列出所用框架、假设、替代解释，明确标注E级 |
| 📊 **量化** | 需要数据统计或结构化产出 | 须给出数据来源、统计方法、样本说明 |
| 🔗 **关联** | 跨模块信息的交叉比对 | 须标注所关联的其他模块编号 |

### 0.4 信息缺口处理机制（Gap Handling Protocol）

每道题的输出须包含三个字段：

```
【已获信息】：（实际采集到的内容与证据等级）
【信息缺口】：（未能获得的信息项）
【缺口原因】：
  □ 法定保密/未公开
  □ 来源存在但受限（付费墙/地理限制/语言障碍）
  □ 来源可能存在但未检索到（建议进一步检索方向：___）
  □ 信息本身不存在（该人物/该时期确无此类记录）
```

## 模块一：身份与履历基础（Identity & Biography）

> **目标**：建立人物的基本事实坐标，形成其他所有模块的锚定层。

### 1.1 姓名与身份标识 📋
- 法定全名（身份证/护照名）、常用别名、英文译名、日文译名、台罗马字拼写
- 每个名称变体的首次公开出现时间与权威来源
- 是否存在改名记录及其时间与原因

### 1.2 出生与成长 📋
- 出生年月日、出生地（县市/乡镇）、成长地
- 户籍地与选区的关系（是否原籍参选、空降）
- 家庭背景：父母职业、族群/省籍背景、社经阶层（公开可得范围内）

### 1.3 教育经历 📋
- 完整学历清单：学校、系所、入学/毕业年份、学位
- 论文/研究题目（如有）、指导教授
- 学历争议点（若有）：争议内容、各方证据、查核结论

### 1.4 进入政坛前的职业经历 📋
- 完整时间线（YYYY-MM 精度），含机构、职位、工作内容
- 关键转换节点：从什么离开、因何进入政治领域
- 该阶段形成的专业能力与人脉网络（与后续政治生涯的关联性标注）

## 模块二：政治生涯与党派动态（Political Career & Party Dynamics）

> **目标**：构建完整的政治履历时间轴，理解其在党内与政治体系中的位置变迁。

### 2.1 公职与头衔完整列表 📋📊
- 所有公职/党职/政治头衔，含：职务名称、起止年月、取得方式（选举/任命/递补）、权责范围
- 按时间排列，标注时期代号

### 2.2 党籍与政治立场变迁 📋
- 入党/退党/改党/停权/处分的完整节点清单
- 每个节点的时间、事件背景、当事人说法、外界评论
- 政治立场自我定位的变化（如：从XX派到XX派）

### 2.3 派系归属与党内山头 📋🔬
- 历任派系归属及变化时间线
- 标志其派系位置改变的关键事件
- 与各派系核心人物的关系性质与变化

### 2.4 关键转折点分析 🔬
- 识别其政治生涯的 **五个关键转折点**
- 每个转折点须包含：
  - 背景（当时的政治环境与个人处境）
  - 行动（其具体决策与行为）
  - 结果（短期与长期后果）
  - 争议（各方评价与反事实推演）
  - 证据链（A/B级来源）

### 2.5 首次进入公众视野 📋
- 关键事件描述、时间、媒体报道
- 当时的采访/声明原文（或最接近的可得版本）
- 该事件对其后续政治路径的塑造作用

## 模块三：核心价值与宪政立场（Core Values & Constitutional Position）

> **目标**：提取其政治理念的底层结构，建立评估其所有政策立场一致性的基准。

### 3.1 自我定义的核心价值观 📋🔬
- 提取 **10条最具代表性的原话**，须附出处（场合、日期、媒体）与上下文
- 归纳其价值体系的核心轴线（如：本土/国际、保守/进步、民粹/菁英等）

### 3.2 国家认同与宪政框架 📋🔄
- 在"国家认同/宪政框架/两岸定位"上的立场表态
- 按时期代号排序，追踪其演变轨迹
- 每条表态须附原文、场景与来源

### 3.3 宪政改革主张 📋
- 对"中华民国体制、修宪、政府体制改革"的立场
- 具体主张内容（如：总统制/内阁制、废考监、修宪门槛等）
- 是否有提案/投票记录佐证

### 3.4 民主法治与司法改革 📋
- 对"司法独立、检调改革、法院体系"的立场
- 推动或反对过的具体法案/措施清单
- 相关发言与投票记录

## 模块四：内政政策立场图谱（Domestic Policy Matrix）

> **目标**：系统覆盖主要内政领域的立场，为"政策立场矩阵"交付物提供原料。

**通用要求**：每个子题须包含——
1. 明确的政策立场表述（附原话与来源）
2. 具体的政策工具/法案/预算主张
3. 实际行动证据（投票、提案、行政命令、预算编列）
4. 立场一致性评估（前后是否矛盾，与党团立场是否一致）

### 4.1 经济总体路线 📋🔬
- 市场vs.国家介入、产业升级、分配正义的基本路线
- 原话与政策文件佐证

### 4.2 财税与财政纪律 📋
- 税制改革、债务管理、预算结构主张

### 4.3 产业政策 📋
- 半导体、AI、关键技术、制造业回流等领域的立场与推动措施

### 4.4 能源政策 📋🔄
- 核能、再生能源、电网韧性、缺电风险的立场
- 立场一致性追踪：不同时期的表态是否矛盾

### 4.5 气候与环境 📋
- 净零承诺、碳费/碳税、ESG政策
- 承诺与落实证据的对照

### 4.6 房市与居住正义 📋
- 房市调控、土地政策、社会住宅、都市更新的主张与实际作为

### 4.7 劳动政策 📋
- 最低工资、工时、派遣、工会相关立场与具体行动

### 4.8 医疗健保 📋
- 健保制度、药价、医护人力、医改主张
- 是否有具体推动记录

### 4.9 长照与人口政策 📋
- 长照体系、老人福利、少子化对策、托育补助方案
- 资金来源说明

### 4.10 教育政策 📋
- 课纲、大学自治、技职教育、双语政策、学费的立场与行动

### 4.11 青年政策 📋
- 就业、薪资、住房、创业相关政策清单

### 4.12 社会福利与贫富差距 📋
- 补助、救助、年金改革立场
- 投票/预算记录

### 4.13 公共安全与治安 📋
- 毒品、诈欺、枪械管制、治安政策立场与行动

### 4.14 交通与公共建设 📋
- 公共运输、道路安全、重大工程的主张与推动项目

### 4.15 公共工程与采购透明 📋
- 标案采购、工程伦理制度主张
- 争议案例

### 4.16 地方治理 📋
- 地方自治、财政分配、区域发展（北中南/离岛）立场与施政主轴

### 4.17 族群与转型正义 📋
- 原住民政策、族群权益、转型正义立场与具体作为

### 4.18 性别与人权 📋
- 性别平等、婚姻平权、家暴与性暴力防治立场
- 投票/发言证据

### 4.19 宗教与社会团体 📋
- 宗教政策、宫庙管理、宗教自由立场
- 与宗教团体的互动网络

### 4.20 文化与语言政策 📋
- 文创、历史记忆、语言政策（台语/客语/原住民语）立场与作为

### 4.21 移民政策 📋
- 新住民、外劳、移民政策主张

### 4.22 数位治理与隐私 📋
- 个资保护、数据开放、AI监管立场
- 推动过的法制/政策

### 4.23 媒体与资讯环境 📋
- 媒体监管、新闻自由、假讯息治理、平台治理立场与政策工具

### 4.24 言论自由与国安平衡 📋
- 言论自由、集会游行权利与国家安全之间的平衡立场

### 4.25 反腐与廉政 📋🔄
- 廉政承诺内容
- 实际行动证据（推动立法、预算、机构改革）
- 承诺与行动的一致性验证

## 模块六：立法与治理绩效（Legislative & Governance Performance）

### 6.7 竞选承诺兑现核查 🔄📊

## 模块十六：综合交付物（Integrated Deliverables）

> **目标**：将前述所有模块的采集与分析成果整合为可直接使用的结构化文件。

### 16.1 可核验人物画像摘要
- **50条事实**（每条一句话）+ 每条对应来源链接与日期 + 证据等级
- 按模块编号索引

### 16.2 争议与事实核查矩阵
- 列表格式：争议编号 / 主张方 / 主张内容 / 支持证据（等级）/ 反证（等级）/ 综合可信度 / 当前状态 / 来源链接

### 16.3 政策立场矩阵
- 列表格式：政策域 / 立场摘要 / 关键原话 / 证据链接 / 时期代号 / 一致性评分（1-5）/ 评分依据
- 一致性评分标准：
  - 5 = 立场明确且言行完全一致
  - 4 = 立场明确，行动大致一致，有轻微偏差
  - 3 = 立场经历过调整但有合理解释
  - 2 = 立场前后矛盾，解释不充分
  - 1 = 立场高度模糊或严重言行不一

### 16.4 盟友—对手—关键组织网络清单
- 每个节点：名称 / 关系类型 / 互动性质 / 时间段 / 关键证据 / 证据等级

### 16.5 整合时间轴
- 格式：YYYY-MM / 事件类型 / 事件描述 / 来源 / 证据等级 / 关联模块
- 标注证据缺口与需进一步查证的问题

### 16.6 数据源索引
- 权威数据源清单：选举结果、立法纪录、预算、财产申报、法院判决、民调
- 每个数据源的入口URL与使用方法
- 学术论文、智库报告、传记、长篇调查报道清单

### 16.7 信息需求清单（Information Requirements List）
- 汇总所有模块中标注为"信息缺口"的条目
- 按优先级排序：对画像结论影响大且有获取可能的排前
- 建议的进一步查证方向
