# 政治人物系统性分析框架（OSINT）

> **版本**: 6.0（全景版）
> **适用对象**: 涉及美国—台湾—中国三角关系的政治人物全景画像与行为预测
> **设计原则**: 事实先行、证据分级、全模块覆盖、行为建模、人格多维交叉验证、网络分析、学术锚定、信息完整性优先、**全景交付导向**
> **学术基础**: Hermann LTA、George操作码、Winter动机分析、Barber类型学、Post远距评估、MBTI认知功能体系、SNA社会网络分析、NLP政治话语分析、文献学方法论、可视化时间序列分析、情景分析学

---

## 设计理念：从V5到V6的范式升级

### V5的基础架构回顾

V5建立了"采集→建模→网络→推演"四层分析架构，并在V4五大行为建模框架基础上，引入了三项核心能力：（1）原始来源全文保留与交叉引用索引（C.8），（2）时间线可视化（D.4），（3）多情景比较分析矩阵（E.5）。V5的核心理念——"分析不遗失原始信息，分析层叠加而非替换"——在V6中被完全继承。

### V6的核心升级理念：**全景覆盖 + 结构化交付**

V6针对V2-V5在实际执行中暴露的两个核心问题：

1. **覆盖率缺口**：V3-V5的范式升级重心在分析建模层（Part C-G），但V2定义的16模块事实采集层在实际报告中往往被不均匀执行——模块一至三、模块八、模块十五覆盖充分，而模块四（25项内政政策）、模块六（立法绩效）、模块七（选举分析）、模块九（两岸专项）、模块十（海外经历）、模块十一（家族财务）、模块十二（争议法律）、模块十三（媒体叙事）、模块十四（三角联动）、模块十六（综合交付物）在实际输出中易被压缩或遗漏。

2. **交付物标准化不足**：V2模块十六定义了7项综合交付物（16.1-16.7），但前几版报告中这些交付物未被系统性地生成并嵌入最终文档。

**V6解决方案**：

- **方案一（全模块显性化）**：在报告文档结构中为V2全部16模块设定显性章节位置，确保每个模块在最终文档中都有对应的分析输出，即使某些模块因信息不足仅能标注"信息缺口"
- **方案二（交付物内嵌）**：将V2模块十六的7项交付物直接内嵌为报告的独立篇章，而非仅作为"附录"处理
- **方案三（覆盖率自检矩阵）**：在方法论声明中增加"V2-16模块覆盖率自检表"，量化每个模块的信息完整度

**核心理念**：*V6 = V2全部16模块的完整事实采集 + V3行为建模五框架 + V4 MBTI认知功能交叉验证 + V5信息完整性原则与时间线可视化 + V6全景交付标准化。每一层叠加而非替换，每一个模块显性化而非隐含。*

## Part A：方法论基础设施（继承V5并强化）

### A.1 证据分级体系

继承V5的A-E五级分级体系及全部框架标注：

| 等级 | 定义 | 典型来源示例 |
|------|------|-------------|
| **A — 官方/法定记录** | 政府机关、法院、立法机关正式文件 | 法院判决书、选委会公告、财产申报表、立法院公报 |
| **B — 权威媒体实名报道** | 具编辑审核流程的媒体机构产出 | 主流日报/通讯社实名报道、具名专访、记者会逐字稿 |
| **C — 可追溯但单一来源** | 可指向具体出处但未经独立交叉验证 | 单一媒体独家、政论节目发言、智库报告（非同行评审） |
| **D — 弱来源/待核实** | 来源模糊、匿名或无法独立验证 | 匿名爆料、论坛传言、未署名社群截图 |
| **E — 分析推断** | 由分析者基于已有证据做出的逻辑推断 | 框架内所有"分析题"的结论 |

**E级框架标注体系**（完整继承V5）：

| 框架标注 | 含义 |
|---------|------|
| [E-Hermann LTA] | 基于Hermann领导特质分析的推断 |
| [E-George OpCode] | 基于George操作码分析的推断 |
| [E-Winter Motive] | 基于Winter动机分析的推断 |
| [E-Barber Type] | 基于Barber类型学的推断 |
| [E-MBTI] | 基于MBTI认知功能分析的推断（须交叉验证） |
| [E-SNA] | 基于社会网络分析的推断 |
| [E-Source] | 来自原始第三方报告的直接事实 |
| [E-Composite] | 基于多框架综合的推断（最高可信度） |

**使用规则**：
- 每条信息必须同时标注【等级】与【来源链接/引用】与【获取日期】
- A/B级信息可直接纳入画像结论
- C级信息须标注"待交叉验证"并标注来源机构立场偏向
- D级信息仅可作为线索列入附录，不可纳入正文结论
- E级推断须明确列出所依据的事实编号、推理逻辑与所使用的学术框架名称

### A.4 信息缺口处理机制

继承V5完整体系（含"分析缺口"类别）：

```
【已获信息】：（实际采集到的内容与证据等级）
【信息缺口】：（未能获得的信息项）
【分析缺口】：（已有信息但尚未完成的分析维度）
  □ 缺乏足够案例建立行为模式（需要N≥3个同类案例）
  □ 缺乏对照组数据（需要同类政治人物的比较基准）
  □ 缺乏时间序列数据（需要跨3个以上时期的纵向追踪）
  □ 学术框架适用性存疑（需要跨文化验证）
【缺口原因】：
  □ 法定保密/未公开
  □ 来源存在但受限（付费墙/地理限制/语言障碍）
  □ 来源可能存在但未检索到（建议进一步检索方向：___）
  □ 信息本身不存在（该人物/该时期确无此类记录）
```

### F.2 写作规范

**厦大战略评估体**写作规范（适用于全篇）：

1. **段落格式**：全文以连贯的学术散文段落呈现，每段150-300字以上，禁止使用项目符号列表
2. **论证结构**：每段遵循"事实铺陈→分析推理→战略含义"的三层递进
3. **语体风格**：采用大陆学术智库的战略评估语体，兼具学术严谨性与政策分析的实操性
4. **证据内嵌**：证据等级标注融入行文（如"根据A级来源……"），不打断阅读节奏
5. **时期标注**：立场演变分析须标注P0-P5时期代号
6. **交叉引用**：跨篇引用使用"参见第X篇§X.X"格式
7. **表格使用**：仅在结构化数据对比时使用表格（如基本信息表、政策矩阵），正文分析不使用表格
