# 一带一路中文教育高完成度补全技能

## 适用场景

当任务目标是：
- 给某个国家补全“一带一路中文教育指标体系”的数据
- 对多个国家做横向比较
- 形成“可交付、可追溯、可继续更新”的证据链
- 尽量一次性做到高完成度，而不是边做边返工

## 不适用场景

- 只想快速写一个概览
- 只需要新闻舆情，不需要教育数据
- 只做一两个字段的临时查询

## 核心原则

### 1. 先收口，再扩展
不要一开始就追 69 个字段的全量终局。

正确做法：
1. 先拿 `P1` 字段做结构判断
2. 再拿 `P2` 字段做数量补强
3. 最后才追 `P3` 的比例/拟合/全国总量难点

### 2. 官方口径优先
证据分三级：

一级证据：
- 教育部
- 国家统计局
- 国家官方年报/白皮书
- 国立大学/公立大学官方院系页
- 官方课程/项目数据库

二级证据：
- 教师协会
- 华教组织
- 研究中心
- 孔子学院

三级证据：
- 新闻报道
- 公开介绍页
- 二手转载

规则：
- 全国总量、统一口径、比例型字段，优先只认一级证据
- 二级证据主要用于补组织网络、培训活动、行业口径
- 三级证据只作线索，不直接回填硬数

### 3. 冲突数字并列，不强行统一
如果官方口径和行业/社群口径冲突：
- 不平均
- 不拍脑袋选一个
- 直接并列
- 强制写清年份、机构、口径

示例写法：
- `MOE 2022：...`
- `协会/社群 2021：...`
- `说明：时间点和统计口径不同`

### 4. 样本值先行，总量随后
对于最难字段，不要一开始就强求全国总量。

先拿：
- 至少几所高校
- 至少几个项目
- 至少多少学员
- 至少几个组织

然后再往全国总量推进。

## 固定工作流

### 第一步：拆任务，不直接在原总表里盲填

先确认工作目录下是否已有：
- `字段字典`
- `国家列表`
- `任务清单`

如果没有，就从原始 `.xls/.xlsx` 母表中拆出：
- 每个字段的定义
- 每个国家的列位
- 每条任务的回填位置

目标不是马上填数，而是把任务变成“可执行单元”。

### 第二步：先形成单国高质量样例

对一个国家，优先拿这些字段：
- 外交伙伴关系等级
- 中文纳入教育体系层级
- 中文语言地位
- 学校/项目/高校/研究机构是否存在
- 教师培训样本
- 学习者样本
- 本地组织/共同体样本

输出：
- 单国态势评估
- 单国汇总数据表
- 单国证据清单

### 第三步：维护三张核心表

后续所有国家都统一维护这三张表：

1. 主任务库
   作用：
   - 记录字段、国家、状态、当前值、来源链接、备注

2. 本地证据表
   作用：
   - 沉淀单条证据
   - 特别是本国语言、本地域名、本地官方页面

3. 核心对照表
   作用：
   - 只保留最关键字段
   - 做多国横向比较

### 第四步：弱项追踪

不要把“还没补完”和“完全没线索”混在一起。

每轮都要更新一张 `弱项追踪表`，至少写清：
- 哪个字段还弱
- 现在已经确认到哪一步
- 为什么还不能写实
- 下一步应该去哪找

## 具体抓取技巧

### 1. 优先“本国语言 + 本地域名”

按国家优先使用：

新加坡：
- `moe.gov.sg`
- `mfa.gov.sg`
- `gov.sg`
- `sccl.sg`
- `ntu.edu.sg`
- `nus.edu.sg`
- `suss.edu.sg`

马来西亚：
- `moe.gov.my`
- `malaysia.gov.my`
- `kln.gov.my`
- `dongzong.my`
- `jiaozong.org.my`
- `utar.edu.my`
- `um.edu.my`

泰国：
- `moe.go.th`
- `mhesi / info.mhesi.go.th`
- `arts.chula.ac.th`
- `hcu.ac.th`
- 其他本地高校官网

印尼：
- `kemdikbud.go.id`
- `pddikti / lldikti`
- `ui.ac.id`
- `binus.ac.id`
- `um.ac.id`

越南：
- `moet.gov.vn`
- `mofa.gov.vn`
- `ulis.vnu.edu.vn`
- `ussh.vnu.edu.vn`
- `ftu.edu.vn`

### 2. PDF / Excel / 年报优先级高于新闻页

优先寻找：
- `Quick Facts`
- `Annual Report`
- `Statistics Digest`
- `Curriculum / Programme`
- `Attachment / Appendix`
- `Download / resource / pdf / xlsx`

### 3. 负面证据也要保留

如果某官方页面明确没有给出目标总量，也要记录。

因为这能支持更稳的结论，例如：
- “本科项目明确存在，但硕士总量未被直接证实”
- “官方统计序列存在，但不是分母表”

### 4. 目录级证据是最后收口的关键

当普通官网页面不再给你新信息时，要切换到：
- 课程目录系统
- 高教数据库
- 招生简章目录
- 项目清单接口

这是从“样本阶段”进入“总量统计阶段”的关键动作。

## 推荐产出物

每做完一个国家或一轮多国推进，至少落这些文件：

- `单国态势评估.md / .docx / .pdf`
- `单国汇总数据.xlsx`
- `本地证据表.xlsx`
- `官方证据表.xlsx`
- `核心对照表.xlsx`
- `弱项追踪表.xlsx`

## 当前最优收口顺序

如果后续继续补别的国家，也建议采用同样顺序：

1. 先打穿制度和结构字段
2. 再补学校数、学习者数、教师数、项目覆盖数
3. 再补比例型和全国总量
4. 最后处理口径冲突与统一说明

## 写入规范

### 强制要求

- 内容必须中文
- 文件名可用稳定 `ASCII`，但表头和内容保持中文
- 不要在几十个版本文件里散写，优先维护“清洁版”主文件

### 强烈建议

- 每次新增证据，都先写进 `本地证据表`
- 每次判断更新，都同步改 `弱项追踪表`
- 每次拿到硬数，立即更新 `核心对照表`

## 完成标准

一个国家达到“高质量可交付”时，至少应满足：

- 有官方制度性证据
- 有学校或项目覆盖样本
- 有学习者或教师样本
- 有高校/研究机构样本
- 有本地组织或共同体样本
- 对尚未补齐的总量字段，已经明确写出：
  - 目前最稳能写到哪一步
  - 为什么还不能写实

## 一句话总结

以后做别的国家，不要再“散着补几十轮”。

正确方式是：
- 先拆任务
- 再做样例
- 固定维护主任务库 + 本地证据表 + 核心对照表 + 弱项追踪表
- 用官方口径优先、样本值先行、冲突口径并列的办法，一次性做到接近收口状态。
