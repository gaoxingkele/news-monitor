## 一、这几轮工作沉淀下来的核心流程

### 1. 先拆总表，再做国家样例
原始总表适合做“指标体系母表”，不适合直接执行抓取。

已经验证更高效的路径是：
- 先拆成 `字段字典`
- 再拆成 `国家列表`
- 再展开成 `任务清单`
- 然后按国家做“样例报告 + 汇总表 + 任务库回填”

优点：
- 不会在大总表里盲填
- 每条证据都能追踪到字段和国家
- 后续增量更新成本低

### 2. 先做能形成态势判断的字段
不要一上来追求 69 个字段一次补齐。

优先顺序已经证明有效：
- `P1`：制度环境、语言地位、纳入教育体系、学校/项目/高校/研究机构是否存在
- `P2`：学校数量、学习者数量、教师数量、培训人数、项目覆盖数
- `P3`：比例型指标、拟合值、舆情比例、总量精算

结论：
- 先拿“结构性字段”做国家态势判断
- 再补“数量型字段”做对比
- 最后再追“全国总量/比例型”难点

### 3. 证据一定分级
后续抓取必须坚持三层证据分级：

- 一级证据：
  - 教育部
  - 国家统计局
  - 国家官方年报/白皮书
  - 国立大学/公立大学官方院系页
  - 官方数据库

- 二级证据：
  - 教师协会
  - 华教组织
  - 孔子学院
  - 研究中心

- 三级证据：
  - 新闻报道
  - 公开介绍页
  - 二手转述材料

规则：
- 全国总量、统一口径、比例型指标，优先只认一级证据
- 二级证据可补“组织网络、培训活动、行业口径”
- 三级证据只作线索，不直接回填硬数

### 4. 同时维护三套表，效率最高
目前证明最好用的结构是：

- `主任务库`
  作用：记录字段、国家、状态、当前值、来源链接、备注

- `本地证据表`
  作用：沉淀单条证据，尤其是本国语言、本地域名、本地官方页面

- `核心对照表`
  作用：做 5 国横向比较，只保留最关键字段

这样做的好处是：
- 主任务库不至于过重
- 增量证据不会散
- 横向对比随时可更新

## 二、已经验证有效的抓取技巧

### 1. 优先“本国语言 + 本地域名”
这条已经非常明确有效。

例如：
- 新加坡：`moe.gov.sg / mfa.gov.sg / gov.sg`
- 马来西亚：`moe.gov.my / malaysia.gov.my / jiaozong.org.my / dongzong.my`
- 泰国：`moe.go.th / arts.chula.ac.th / hcu.ac.th`
- 印尼：`kemdikbud.go.id / pddikti / UI / BINUS`
- 越南：`moet.gov.vn / ulis.vnu.edu.vn / ussh.vnu.edu.vn`

经验：
- 本国语言关键词命中率明显高于英文泛搜
- 本地高校和教育主管部门页面，最容易直接给出项目层级、培训人数、课程口径

### 2. 对“总量”先拿最小可核验值
对最难的字段，不要一开始就强求全国最终总量。

先拿：
- 至少几所高校
- 至少几个项目
- 至少多少名学员
- 至少几个组织

这种“最小可核验值”已经足够支撑高质量判断，而且比写虚高数字安全得多。

### 3. 发现冲突数字时，不合并，直接并列
这是这次特别重要的经验。

典型例子：
- 马来西亚独中数量
  - `MOE`: 60 所
  - `董总`: 63 所

正确做法：
- 同时保留
- 明确写时间和口径
- 不擅自统一成一个数字

### 4. 负面证据也要收集
有些最有价值的进展，不是“找到一个数字”，而是确定：
- 某官方页没有给这个数字
- 某项目不是中文硕士，只是东亚研究硕士
- 某页面不是中文专业，只是应用语言学

这类“否定式证据”能避免误填，非常重要。

### 5. 乱码问题的处理方式
当前环境里，中文文件名和中文 sheet 名容易在某些脚本/终端环节被污染。

已经验证可行的解决方式：
- 文件内容保持中文
- 文件名用稳定 ASCII 或 `ascii + cn` 风格
- sheet 名用稳定英文，表头保持中文
- 最终汇报材料再保留中文名称

## 五、后续继续抓的整体方案

### 方案A：官方总量收口模式
只做：
- 教育部年报
- 国家统计表
- 国立大学项目目录
- 官方数据库

适合字段：
- 学校总量
- 学习者总量
- 教师总量
- 比例型字段
- 硕博总量

### 方案B：本地组织补强模式
只做：
- 教师协会
- 华教组织
- 研究中心
- 专业共同体

适合字段：
- 协会数量
- 培训网络
- 组织结构
- 行业口径

### 方案C：双口径并列模式
当官方与社群数字冲突时：
- 不合并
- 不平均
- 直接并列
- 强制写时间点和来源

模板写法：
- `MOE 2022：...`
- `协会/社群 2021：...`
- `说明：两者统计口径不同`
